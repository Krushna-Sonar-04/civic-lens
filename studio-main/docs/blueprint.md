# **App Name**: Civic Lens

## Core Features:

- Issue Reporting: Allow citizens to report civic issues through a form with image upload, location, and description. This will use the phone's native GPS tool to populate location automatically.
- Issue Display: Display reported issues on a map and in a list format for citizens to view and upvote. Each issue will show its status and details.
- Contractor Task Management: Enable contractors to view assigned tasks, update their status, and upload completion proofs. Availability status can also be toggled.
- Admin Issue Oversight: Provide administrators with a live map view of all reported issues, along with filtering and sorting options. Also, there should be ways to analyze reported issue metrics.
- Automatic Task Assignment: Use an AI tool to assign reported issues to contractors based on location, skill, and current load, automatically.
- Citizen Issue Verification: Verify the classification and information from a newly submitted issue report.

## Style Guidelines:

- Primary color: Deep sky blue (#3EB489) to evoke trustworthiness and civic responsibility.
- Background color: Light gray (#F0F4F7), offering a clean and neutral backdrop.
- Accent color: Yellow-orange (#E99D1A), a contrasting analogous color to highlight calls to action.
- Body and headline font: 'PT Sans', a modern and readable sans-serif font for headlines or body text.
- Use consistent, clear icons to represent issue categories and actions. Icons should be simple and easily recognizable.
- Employ a card-based layout for issue listings to maximize responsiveness and clarity on various screen sizes.
- Use subtle transitions and animations for status changes and loading states to provide feedback without overwhelming the user.
